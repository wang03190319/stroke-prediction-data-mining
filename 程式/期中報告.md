# Stroke Prediction Dataset 資料探勘期中報告

## 一、研究主題與目的

本研究使用 `healthcare-dataset-stroke-data.csv` 醫療資料集進行中風預測分析。資料集中包含病患的基本人口特徵、健康狀況、生活型態與是否曾中風等欄位。本作業的主要目標是透過資料探勘流程，建立可預測中風風險的分類模型，並比較不同模型在不平衡資料下的表現。

由於中風屬於高風險醫療事件，若模型漏判高風險個案，可能造成嚴重後果。因此本研究除了觀察整體分類表現外，也特別重視 `Recall`，也就是模型能找出多少真正中風個案。

## 二、資料集說明

本資料集共有 5110 筆資料與 12 個欄位，目標變數為 `stroke`，其中 `stroke=1` 代表曾發生中風，`stroke=0` 代表未中風。

主要欄位如下：

| 欄位 | 說明 |
| --- | --- |
| `id` | 病患編號 |
| `gender` | 性別 |
| `age` | 年齡 |
| `hypertension` | 是否有高血壓 |
| `heart_disease` | 是否有心臟病 |
| `ever_married` | 是否曾婚 |
| `work_type` | 工作類型 |
| `Residence_type` | 居住地類型 |
| `avg_glucose_level` | 平均血糖值 |
| `bmi` | 身體質量指數 |
| `smoking_status` | 吸菸狀態 |
| `stroke` | 是否中風，為本研究預測目標 |

資料檢查後發現，`bmi` 欄位有 201 筆缺失值，缺失比例為 3.93%。此外，目標變數存在明顯類別不平衡：未中風資料有 4861 筆，佔 95.13%；中風資料只有 249 筆，佔 4.87%。

## 三、資料前處理

在建模前，先移除不具預測意義的 `id` 欄位，避免模型學到單純識別碼造成干擾。接著將資料分成特徵變數 `X` 與目標變數 `y`。

數值欄位包含：

`age`、`hypertension`、`heart_disease`、`avg_glucose_level`、`bmi`

類別欄位包含：

`gender`、`ever_married`、`work_type`、`Residence_type`、`smoking_status`

前處理方式如下：

| 資料類型 | 處理方式 |
| --- | --- |
| 數值欄位 | 使用中位數補值 |
| 類別欄位 | 使用眾數補值，並進行 One-Hot Encoding |
| 資料切分 | 80% 訓練集、20% 測試集 |
| 切分設定 | 使用 `stratify=y` 保持中風與非中風比例一致 |

切分後訓練集共有 4088 筆，測試集共有 1022 筆。

## 四、探索性資料分析

從類別分布可以看出，本資料集的中風樣本比例非常低，屬於典型的不平衡分類問題。如果只看 Accuracy，模型可能偏向預測大多數類別，也就是「未中風」，因此本研究選擇 Recall、F1-score 與 ROC-AUC 作為主要評估指標。

在視覺化分析中，程式分別觀察了：

1. `stroke` 類別分布
2. `age` 與 `stroke` 的關係
3. `avg_glucose_level` 與 `stroke` 的關係
4. `smoking_status` 與 `stroke` 的比例關係

從分析方向來看，年齡、平均血糖、BMI、既有疾病與吸菸狀態都可能與中風風險有關。其中年齡與血糖等數值型變數在後續模型特徵重要度中也呈現較高影響力。

## 五、模型建立與比較

本研究建立兩個分類模型：

| 模型 | 重要設定 |
| --- | --- |
| Logistic Regression | `class_weight='balanced'`、`max_iter=2000` |
| Random Forest | `n_estimators=400`、`class_weight='balanced_subsample'` |

因為資料集中中風樣本比例偏低，因此兩個模型皆加入類別權重設定，希望減少模型過度偏向多數類別的情況。

模型比較結果如下：

| 模型 | Recall | F1-score | ROC-AUC |
| --- | ---: | ---: | ---: |
| Logistic Regression | 0.8000 | 0.2360 | 0.8437 |
| Random Forest | 0.0000 | 0.0000 | 0.7829 |

從結果可以看出，Logistic Regression 在 Recall 上明顯優於 Random Forest。Random Forest 雖然 ROC-AUC 仍有 0.7829，但在預設分類門檻下沒有成功預測出中風類別，因此 Recall 與 F1-score 皆為 0。若以醫療風險篩檢為目標，Logistic Regression 較適合作為本次分析的最佳模型。

## 六、最佳模型評估

本研究選擇 Logistic Regression 作為最佳模型，其測試集結果如下：

| 指標 | 數值 |
| --- | ---: |
| Recall | 0.8000 |
| F1-score | 0.2360 |
| ROC-AUC | 0.8437 |
| Accuracy | 0.7466 |

混淆矩陣如下：

| 實際 / 預測 | 預測未中風 | 預測中風 |
| --- | ---: | ---: |
| 實際未中風 | 723 | 249 |
| 實際中風 | 10 | 40 |

從混淆矩陣可知，測試集中實際中風者共有 50 筆，其中模型成功找出 40 筆，漏判 10 筆，因此 Recall 達到 0.80。不過模型也將 249 筆未中風者判為中風，使得 Precision 偏低。這代表模型偏向提高敏感度，適合初步風險篩檢，但若要實際用於醫療決策，仍需要搭配後續檢查與更嚴謹的模型調整。

分類報告摘要如下：

| 類別 | Precision | Recall | F1-score | Support |
| --- | ---: | ---: | ---: | ---: |
| 未中風 `0` | 0.9864 | 0.7438 | 0.8481 | 972 |
| 中風 `1` | 0.1384 | 0.8000 | 0.2360 | 50 |

## 八、結論

本研究使用中風預測資料集完成資料載入、缺失值處理、探索性分析、模型建立與模型評估。資料集中 `stroke=1` 的比例僅 4.87%，存在嚴重類別不平衡，因此本研究以 Recall 作為主要評估指標。

在 Logistic Regression 與 Random Forest 的比較中，Logistic Regression 表現較佳，在測試集上達到 0.8000 的 Recall 與 0.8437 的 ROC-AUC。這表示模型能找出大部分真正中風的個案，適合作為初步風險篩檢的方向。不過 Precision 僅 0.1384，代表誤報比例偏高，仍有改善空間。

整體而言，本次分析顯示年齡、平均血糖、BMI、高血壓、心臟病與生活型態欄位對中風預測具有一定貢獻。未來若要提升模型實用性，可以進一步嘗試調整分類門檻、使用 SMOTE 等過採樣方法、加入交叉驗證，或比較 XGBoost、LightGBM 等模型，以改善不平衡資料下的分類效果。
